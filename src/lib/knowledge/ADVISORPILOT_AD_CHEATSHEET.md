## Pre-flight checklist (30 seconds)

- [ ] I am selling **workflow software**, not investment advice or returns
- [ ] I know which **content pillar** this ad supports
- [ ] I can state the **before → after transformation** in one line
- [ ] Copy uses **AdvisorPilot™** (trademark) on first brand mention
- [ ] No **em-dashes** anywhere in copy
- [ ] No **prohibited claims** (performance, SEC approval, AI replaces judgment)
- [ ] **Disclaimer** is included on the creative
- [ ] Product UI shown matches **real AdvisorPilot** (navy rail, wizard steps, square corners)
- [ ] QR is **prominent** (top-right), not a tiny corner afterthought

## What AdvisorPilot is

**AdvisorPilot** is purpose-built **workflow software for independent financial advisors and small ensembles**. It connects custodian statements, internal analysis, and client-ready materials in one continuous sequence.

| AdvisorPilot IS | AdvisorPilot IS NOT |
|-----------------|---------------------|
| Portfolio **review operations** software | A robo-advisor or trading platform |
| Statement intake → analysis → deliverables | CRM-only or document storage |
| AI-assisted **workflow preparation** | AI that gives investment advice |
| Built for RIAs who need operational discipline | Consumer fintech or hype-driven SaaS |
| Wirehouse-grade process without wirehouse headcount | A guarantee of returns or performance |

**One-sentence pitch:** Operational discipline for portfolio reviews, without adding headcount.

**Website:** https://www.advisorpilot.ai/

## The problem we solve

Reviews break at the **handoffs**, not in the meeting:

1. Custodian PDFs arrive in email
2. Positions get copied into spreadsheets
3. Narrative gets drafted in haste the night before
4. Three versions of the same account get reconciled manually
5. Supervision and client communication happen in disconnected tools

**Core insight for copy:** The prep took longer than the advice. The meeting is rarely the bottleneck.

## How the product actually works

### The 10-step review workflow

AdvisorPilot's hero surface is a guided wizard (`/app/intake`). Steps in order:

| Step | UI name | What happens |
|------|---------|--------------|
| 1 | Client Profile | Intake form — household, objectives, registration context |
| 2 | Upload | Custodian statement upload (advisor or client magic link) |
| 3 | Confirm | Holdings reconciliation — verify positions before analysis |
| 4 | Analysis | Risk, diversification, income scores; donuts, stress scenarios, insights |
| 5 | Meeting guide | Walkthrough rail with opening/closing scripts, talking points |
| 6 | Roth worksheet | Roth conversion calculator |
| 7 | FIA calculator | Fixed income annuity analysis |
| 8 | Ret. Inc Calculator | Retirement income projection |
| 9 | Comparative fee analysis | Fee comparison worksheet |
| 10 | Client Review Report | PDF preview, download, email — client-ready deliverables |

### Three layers beneath every review (marketing framework)

Use this structure when explaining value:

1. **Data foundation** — Structured intake, registration context, confirmed book of record
2. **Analysis & narrative** — Drafted themes, scenarios, meeting prep the team edits
3. **Governed outputs** — Materials and correspondence ready for internal approval

### Three workflow pillars (website hero cards)

| Pillar | Message |
|--------|---------|
| **Ingest & confirm** | Custodian statements via advisor upload or secure client link. Reconcile positions before analysis depends on them. |
| **Analyze & narrate** | Structure risk, diversification, and scenario context into drafts your team edits. Meetings start from one narrative base. |
| **Document & follow up** | Package materials for supervisory review. Release through channels and disclosures you already use. |

### CRM modules (secondary surfaces)

- **CRM** — Client roster, profile KPIs (AUM, YTD, accounts), tabs: Overview, Portfolio, Workflow, Notes, Tasks, Documents
- **Tasks** — Firm-wide task management
- **Reports** — Report library and viewer
- **Flight Deck** — AI-assisted presentation decks
- **QR Code** — Client magic-link upload (clients upload statements from phone, no login)
- **Nova** — In-app AI chat + optional voice assistant

### Client magic link (important differentiator)

Advisors create a secure link (or QR). Clients confirm intake and upload statements from their phone. Row appears as **Draft** in client database. This is real product — good for "statement intake" messaging.

### AI positioning (critical)

- AI **structures** holdings, risk, diversification into **drafts**
- AI **assists operational prep** — does **not** replace fiduciary judgment
- **Nothing is transmitted without advisor action**
- Advisor **edits before anything is shared externally**
- Separation of **operational preparation** from **fiduciary advice**

## Copy guardrails (hard rules)

1. **No em-dashes** — use commas or periods (`sanitizeNoEmDash` enforces this)
2. **AdvisorPilot™** trademark on brand mentions
3. **Standard disclaimer** on every creative:

   > For professional workflow and discussion only. Not an offer, solicitation, or recommendation of securities or advisory services. AI assists workflow preparation, not investment advice.

4. Headlines: max ~90 chars, aim for **one clear message**, intentional line breaks with `\n`
5. Subheads: max ~140 chars, quieter than headline
6. **Transformation line** required: `before → after` format
7. CTA on social: **QR code** is the primary action — label "Request a demo" under QR, not a separate button competing for attention

## Legal review triggers (RIA marketing rules)

The Creative Studio pipeline auto-checks for:

- Performance guarantees and return figures
- Market outperformance claims
- Testimonials without disclosures
- Implied investment advice
- False SEC/FINRA endorsement
- AI advice overclaims ("replaces judgment," "automated advice")
- Superlative rankings (#1, best ever)

If flagged, the fix loop rewrites copy — stay inside guardrails from the start.

## Storytelling framework for every ad

Every ad must communicate **transformation**, not "there is software."

```
BEFORE (pain)          →    AFTER (outcome)
─────────────────────────────────────────────
Hours on prep          →    Minutes to deliverables
Manual PDF parsing     →    Structured holdings
Dense spreadsheets     →    Clear client narratives
Disconnected systems   →    One unified platform
Manual analysis        →    Auditable AI workflow
```

Structure:
1. **Headline** — the after-state or the tension (editorial, 2 lines max)
2. **Transformation** — explicit before → after
3. **Subhead** — one line of how (the workflow)
4. **Product UI** — prove it's real
5. **QR** — scan to request demo
6. **Disclaimer** — legal, quiet

## Common mistakes (do not repeat)

| Mistake | Fix |
|---------|-----|
| Logo overlaps headline | Grid layout: header row → copy row → product row |
| Blank top-right corner | Large QR code lives there |
| Tiny useless QR in corner | 152px+ QR top-right with "Request a demo" label |
| "Request a demo" button + QR | QR only — one CTA |
| Generic placeholder UI cards | Real AdvisorPilot shell: navy rail, wizard rail, square corners |
| Copying reference ad PNGs literally | Reference ads were inspiration, not templates |
| Feature lists on the card | One message, one transformation |
| Em-dashes in copy | Commas or periods only |
| Promising investment outcomes | Workflow benefits only |
| Missing AdvisorPilot™ | Trademark every brand mention |
| Flat white background | Warm off-white with subtle ambient gradient |
